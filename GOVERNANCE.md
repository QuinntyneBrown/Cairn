# Governance

## Purpose

This document describes how Cairn is maintained and how project decisions are made.

## Roles

### Maintainers

Maintainers are responsible for:

- Setting technical direction
- Reviewing and merging pull requests
- Managing releases and repository settings
- Enforcing the Code of Conduct and security policy

### Contributors

Contributors are responsible for:

- Proposing and implementing improvements
- Following repository contribution guidelines
- Keeping discussions constructive and evidence-based

## Decision making

- Routine technical decisions are made through pull-request review and maintainer approval.
- Significant changes should begin with an issue that documents context, options, and tradeoffs.
- When consensus cannot be reached, maintainers make the final decision and document rationale in the issue, pull request, or ADR.

## Becoming a maintainer

Maintainers may invite active contributors who consistently demonstrate:

- High-quality technical judgment
- Reliability in reviews and follow-through
- Respectful collaboration with the community
- Stewardship of project quality and direction
